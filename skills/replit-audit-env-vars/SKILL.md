# Env-Var Contract Audit

Builds the complete environment-variable contract by exhaustive grep, cross-references each variable against its callers, and produces the migration mapping (where it lives on Replit → where it should live on the target platform).

## The deterministic four-grep recipe

These four greps produce the union of every env-var the codebase reads:

```bash
# Server-side / scripts (process.env)
grep -rhE "process\.env\.[A-Z_]+" server/ scripts/ shared/ 2>/dev/null \
  | grep -oE "process\.env\.[A-Z_]+" | sort -u

# Client-side / Vite-injected (import.meta.env)
grep -rhE "import\.meta\.env\.[A-Z_]+" client/ shared/ 2>/dev/null \
  | grep -oE "import\.meta\.env\.[A-Z_]+" | sort -u

# Any string literal starting with VITE_ (catches dynamic accesses)
grep -rhE "VITE_[A-Z_]+" . 2>/dev/null | grep -v node_modules \
  | grep -oE "VITE_[A-Z_]+" | sort -u

# Case-insensitive 'replit' (catches uppercase REPLIT_*, lowercase replit_*, etc.)
grep -riE "replit" . 2>/dev/null | grep -v node_modules | head -50
```

Compile the union into a single env-var list. Deduplicate.

## For each variable, document

Build a table with these columns:

| Column | What goes in it |
| --- | --- |
| Variable name | e.g., `DATABASE_URL` |
| Source on Replit | auto-provisioned / `[secrets]` block / `[userenv.*]` block / hardcoded fallback |
| Suggested portable source | Secret Manager / plain env var / generated by Terraform random_password / etc. |
| Active callers | file paths + line numbers where the variable is read |
| Default / fallback | what value the code uses if the env var is missing (`process.env.X \|\| "default"`) |
| Severity | Critical (auth/encryption/DB) / Standard (API keys) / Non-secret (URLs, region names) |

The "Active callers" column is essential — variables with zero callers can be removed; variables with multiple callers need to maintain consistency across them.

## Three catches specific to Replit-origin projects

### Catch 1 — Non-DB uses of `DATABASE_URL`

Replit's auto-provided `DATABASE_URL` is always present, so developers sometimes reuse it as a convenient key source for non-DB purposes:

```bash
grep -rE "DATABASE_URL" server/ shared/ | grep -v -E "(Pool|client|drizzle|pg\.)" | head
```

Common offenders: HMAC seeds for share tokens, dev-fallback strings for other secrets, cache namespacing. These break the moment `DATABASE_URL` rotates or contains an unexpected character (e.g., a new password with `?` or `#`).

Document each non-DB usage. Recommend deriving a separate purpose-specific secret instead.

### Catch 2 — `VITE_*` variables baked into the bundle

`import.meta.env.VITE_X` reads are inlined into the client bundle at build time. Any `VITE_*` variable that contains a credential is **publicly leaked** once the bundle ships. Cross-reference each `VITE_*` finding against the credentials checklist:

- `VITE_*` containing `SECRET`, `KEY`, `TOKEN`, `PASSWORD` → flag as `VLN` (likely VLN-002 family)
- `VITE_*` containing URLs, feature flags, public client IDs → safe

### Catch 3 — Replit's `REPL_ID`, `REPL_SLUG`, `REPL_OWNER` as gating conditions

These three env vars are Replit-provided. Code that does `if (process.env.REPL_ID) { ... }` gates Replit-specific behavior. Document each gating site — the *gate itself* is fine (no-ops off-Replit), but the code inside the gate often references Replit-specific services that won't have non-Replit equivalents.

## What to write to the audit document

```markdown
## 3. Environment variable contract

### All variables (N total)
| Variable | Replit source | Portable source | Callers | Default | Severity |
| --- | --- | --- | --- | --- | --- |
| `DATABASE_URL` | auto | Cloud SQL Auth Proxy URL | server/db.ts, scripts/db-migrate.js | none | Critical |
| `SESSION_SECRET` | `[secrets]` | Secret Manager (Terraform-generated) | server/auth.ts | `${DATABASE_URL}` (bad — see catch 1) | Critical |
| `VITE_DEMO_MODE` | `[userenv.development]` | plain env on Cloud Run | client/login.tsx | `false` | Non-secret |
| `REPL_ID` | Replit-provided | n/a (gate only; not used off-Replit) | client/main.tsx (cartographer gate) | undefined | Non-secret |
| … |

### Non-DB uses of DATABASE_URL flagged
- server/share-tokens.ts:42 uses `crypto.createHmac("sha256", DATABASE_URL)` for share-token signing. Recommend extracting into a separate `SHARE_TOKEN_SIGNING_KEY` secret.

### VITE_* variables that contain credentials
- (none / VLN-NNN: VITE_ADMIN_PASSWORD baked into client bundle)
```

## What this skill doesn't do

- Doesn't decide *which* portable secret system to use (Secret Manager vs Doppler vs platform-native). That's a Phase 3 decision driven by platform selection.
- Doesn't rotate or change any values. Those happen in `replit-localize-security-cleanup`.

## Parent skill

Invoked from `replit-audit`. The env-var contract is the most-referenced output of the audit — `replit-to-gcp-terraform-secrets` consumes it directly to decide which secrets need Terraform-managed entries.
